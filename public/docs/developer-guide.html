WMG YouTube Tracker v2

Hệ thống theo dõi views YouTube, phân tích hiệu suất nhân sự, và tính lương theo kênh.

Giới thiệu hệ thống

Ứng dụng quản lý kênh YouTube cho team nội dung — từ tracking views, phân tích nhân sự đến tính lương tự động.

3 Roles

DIRECTOR, MANAGER, STAFF — mỗi role có quyền truy cập và tính năng riêng.

Multi-channel

Mỗi kênh YouTube được OAuth riêng, sync dữ liệu độc lập.

Analytics Sync

Pull dữ liệu từ YouTube Analytics API v2, lưu vào PostgreSQL theo date range.

Database & Connection

PostgreSQL hosted trên Supabase, kết nối qua Prisma với @prisma/adapter-pg.

// lib/db/index.ts
import { PrismaClient } from "@prisma/client"
import { PrismaPg } from "@prisma/adapter-pg"
import { Pool } from "pg"

// Singleton pattern — tránh tạo nhiều connection trong hot reload
const pool = new Pool({ connectionString: process.env.DATABASE_URL })
const adapter = new PrismaPg(pool)
export const db = new PrismaClient({ adapter })

Commands

npm run db:migrate
npm run db:push
npm run db:studio
npm run db:seed

Supabase lưu ý

Dùng DATABASE_URL (pooled) cho runtime, DIRECT_URL (direct) cho migrations. Nếu dùng cùng 1 URL sẽ lỗi migration timeout.

Payroll Models

ChannelWeightConfig → channel_weight_configs

Tỷ lệ phân chia views theo role. VD: WRITER = 40%, EDITOR = 60%.

Unique: (channelId, role, effectiveFrom) — có thể thay đổi theo thời gian.

SalaryConfig → salary_configs

Cấu hình lương cho từng nhân sự: baseSalary + bonusPerThousandViews.

Có lịch sử (effectiveFrom) — tính lương đúng kỳ.

PayrollPeriod → payroll_periods

Kỳ lương (tháng/năm). Unique (month, year). Có thể lock để tránh thay đổi.

PayrollRecord → payroll_records

Kết quả tính lương: baseSalary + totalBonus = totalSalary. Lưu detail JSON để audit.

Flow — Authentication

1
User nhập email + password tại /login
↓
2
NextAuth CredentialsProvider verify bcrypt hash từ DB
↓
3
JWT được tạo chứa { id, email, role }
↓
4
Redirect đến /dashboard → server component đọc session
↓
5
Redirect theo role: DIRECTOR → /director/analytics, MANAGER → /manager/analytics, STAFF → /staff/analytics

Flow — YouTube Analytics Sync (Production / Netlify)

1
User bấm "Cập nhật từ YouTube" trên UI
↓
2
POST /api/analytics/sync — tạo SyncJob (status: "pending"), trả về { jobId } ngay lập tức (HTTP 202)
↓
3
Server fire-and-forget đến /.netlify/functions/sync-background với jobId + secret
↳ Netlify Background Function chạy tối đa 15 phút, không block response
↓
4
Background function cập nhật job → status: "running"
↓
5
Gọi syncAnalyticsSnapshots(): lấy token → gọi YouTube Analytics API v2 → upsert AnalyticsSnapshot
↓
6
Cập nhật job → status: "done" với result JSON
↓
7
Client (SyncButton) poll GET /api/analytics/sync/status?jobId=xxx mỗi 3 giây → nhận "done" → refresh data

Flow — Local Dev Sync

Khi process.env.NETLIFY không được set, sync chạy inline trong Node.js.

1
POST /api/analytics/sync — tạo SyncJob, detect !NETLIFY
↓
2
Trả về { jobId } HTTP 202 ngay, đồng thời khởi động async IIFE (fire-and-forget)
↓
3
IIFE update job → "running" → gọi syncAnalyticsSnapshots() → update "done"
↓
4
Client poll như bình thường cho đến khi nhận "done"

Flow — Tính Lương

1
Director tạo PayrollPeriod (tháng/năm)
↓
2
POST /api/payroll/periods/[id]/calculate — đọc tất cả VideoRoleAssignment (APPROVED) trong kỳ
↓
3
Tính views cho mỗi video: AnalyticsSnapshot.views cho date range của kỳ
↓
4
Phân chia theo role weight: WRITER nhận 40% views/video, EDITOR nhận 60% / số editors
↓
5
Tính bonus: totalViews / 1000 × bonusPerThousandViews
↓
6
Tạo PayrollRecord cho từng nhân sự với detail JSON đầy đủ
↓
7
Director lock kỳ lương → không thể thay đổi sau đó
