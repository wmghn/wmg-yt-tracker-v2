Cài đặt Netlify CLI & plugin Next.js

Plugin @netlify/plugin-nextjs là bắt buộc để Next.js 14 App Router hoạt động đúng trên Netlify (SSR, API Routes, Middleware).

Cài plugin vào project

npm install --save-dev @netlify/plugin-nextjs

File netlify.toml đã được tạo sẵn trong project

File này đã cấu hình plugin, security headers và cache. Không cần chỉnh thêm.

ℹ️
Netlify tự detect plugin — Nếu netlify.toml đã khai báo [[plugins]], Netlify sẽ tự install khi build. Bạn không cần cài thủ công trên dashboard.
2

Kết nối repository GitHub

  1. Đăng nhập app.netlify.com → nhấn "Add new site" → "Import an existing project".
  2. Chọn GitHub → cấp quyền → chọn repo wmg-yt-tracker-v2.
  3. Netlify tự đọc netlify.toml — branch deploy mặc định là main.
  4. Chưa nhấn Deploy — đi đến Bước 4 để điền biến môi trường trước.
⚠️
Điền env TRƯỚC khi deploy lần đầu — Nếu build chạy mà thiếu DATABASE_URL hay NEXTAUTH_SECRET, app sẽ crash.

Thiết lập Environment Variables Quan trọng nhất

Cách vào trang Environment Variables

  1. Trên dashboard Netlify, chọn site của bạn.
  2. Menu bên trái: Site configuration → Environment variables.
  3. Nhấn "Add a variable" → chọn "Add a single variable".
  4. Điền Key và Value → nhấn "Save".
  5. Lặp lại cho từng biến trong bảng ở Bước 5.
🔴
KHÔNG BAO GIỜ làm những điều này:
  • Commit file .env hoặc .env.production lên Git
  • Dán secret vào netlify.toml (file này đã ở trong repo)
  • Chia sẻ NEXTAUTH_SECRET hay DATABASE_URL qua Slack/email dạng plain text
  • Dùng chung secret giữa môi trường dev và production

Scope phù hợp cho từng biến

Netlify cho phép chọn scope: Production Deploy Previews Branch deploys Local development.

→ Với các secret nhạy cảm (DATABASE_URL, NEXTAUTH_SECRET, ENCRYPTION_KEY), chỉ chọn Production để không lộ ra preview branches.

💡
Tip: Dùng database riêng cho production (Supabase project khác hoặc Neon branch khác). Không bao giờ để dev và production dùng chung DB.

Cách tạo SECRET an toàn

🔴
KHÔNG dùng các chuỗi tự gõ như mysecret123 — phải dùng random generator.

Tạo NEXTAUTH_SECRET

Chạy lệnh này trên terminal (máy local hoặc bất kỳ máy nào có Node/OpenSSL):

# Cách 1 — OpenSSL (khuyến nghị)
openssl rand -base64 32

# Cách 2 — Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"

# Cách 3 — Online (chỉ dùng khi trust site)
# https://generate-secret.vercel.app/32

Tạo ENCRYPTION_KEY (64 hex chars)

# OpenSSL
openssl rand -hex 32

# Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

Tạo CRON_SECRET

openssl rand -base64 24
⚠️
Lưu trữ secrets an toàn:
  • Dùng 1Password, Bitwarden hoặc Notion (private, encrypted) để lưu.
  • Không lưu trong Google Docs, Slack, hay ghi chú điện thoại không có mã hoá.
  • Sau khi điền vào Netlify, xoá khỏi lịch sử terminal: history -c

Cron Job — Tự động Sync Views

Project đang dùng Vercel Cron (vercel.json) để tự gọi /api/cron/sync-views mỗi 2 ngày. Netlify không hỗ trợ Vercel Cron, có 2 lựa chọn thay thế:

Lựa chọn 1 — Dùng cron-job.org (Miễn phí, đơn giản)

  1. Đăng ký tại cron-job.org (miễn phí).
  2. Tạo job mới: URL = https://ten-site.netlify.app/api/cron/sync-views
  3. Schedule: 0 2 */2 * * (2 giờ sáng, mỗi 2 ngày).
  4. Thêm header: Authorization: Bearer <CRON_SECRET>

Lựa chọn 2 — GitHub Actions (Miễn phí, có log rõ ràng)

Tạo file .github/workflows/sync-views.yml:

name: Sync Views
on:
  schedule:
    - cron: "0 2 */2 * *"   # 02:00 UTC mỗi 2 ngày
  workflow_dispatch:          # Cho phép chạy thủ công

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - name: Call sync endpoint
        run: |
          curl -f -X GET \
            -H "Authorization: Bearer ${{ secrets.CRON_SECRET }}" \
            https://ten-site.netlify.app/api/cron/sync-views

Vào GitHub repo → Settings → Secrets → Actions → thêm secret CRON_SECRET.

ℹ️
Khuyến nghị: GitHub Actions vì có log đầy đủ, dễ debug, miễn phí 2000 phút/tháng trên repo public/private.

Kiểm tra sau khi Deploy

  1. Vào https://ten-site.netlify.app/login → đăng nhập với tài khoản Director.
  2. Kiểm tra menu: Quản lý Kênh, Nhân sự, Thống kê — phải load được.
  3. Mở DevTools → tab Network → gọi 1 API (ví dụ /api/analytics/staff-analytics) → response phải là JSON, không phải lỗi 500.
  4. Kiểm tra security headers: mở https://securityheaders.com → nhập URL site → phải đạt grade A trở lên.
  5. Kiểm tra SSL: URL phải bắt đầu bằng https:// và không có cảnh báo certificate.
  6. Test cron endpoint thủ công:
    curl -H "Authorization: Bearer <CRON_SECRET>" https://ten-site.netlify.app/api/cron/sync-views

Nếu build thất bại

Vào Netlify dashboard → Deploys → [deploy thất bại] → Deploy log để xem lỗi chi tiết.

Lỗi thường gặp:

LỗiNguyên nhân & Fix
Environment variable not found: DATABASE_URL Chưa thêm biến vào Netlify. Vào Site configuration → Environment variables → thêm lại → Redeploy.
Prisma Client is not generated Thêm script: "build": "prisma generate && next build" vào package.json.
NEXTAUTH_URL is not set Thêm biến NEXTAUTH_URL với URL chính xác của Netlify site.

Trước khi share link site cho người dùng, kiểm tra lại:

Hạng mụcTrạng thái

NEXTAUTH_SECRET là chuỗi random ≥ 32 ký tự, khác với dev □ Kiểm tra thủ công
NEXTAUTH_URL trỏ đúng URL production (không phải localhost) □ Kiểm tra thủ công

YouTube API Key đã restrict: chỉ cho domain Netlify của bạn □ Kiểm tra trên Google Cloud Console
Endpoint /api/cron/sync-views yêu cầu header Authorization ✓ Đã có trong code
Security headers được trả về (X-Frame-Options, HSTS...) ✓ Đã cấu hình trong netlify.toml
Header X-Powered-By: Next.js đã bị ẩn ✓ Đã tắt trong next.config.mjs

Secrets được lưu trong password manager, không trong ghi chú tự do □ Kiểm tra thủ công
